## Description

Red Hat Satellite 6.14 server SATELLITE-PROD-01 is failing to sync content from the Red Hat CDN for RHEL 8 BaseOS and AppStream repositories. Error: 'Metadata checksum mismatch'. Monthly patching cycle starts in 3 days and content must be current.

## Resolution Notes

ROOT CAUSE ANALYSIS:
The Satellite server's /var/lib/pulp partition ran out of space (100% utilization) during a previous partial sync, leaving corrupted/incomplete metadata files in the Pulp content store. Subsequent sync attempts failed checksum validation against the CDN because the local metadata was incomplete.

RESOLUTION STEPS TAKEN:
1. Checked Satellite task details in the WebUI → Monitor → Tasks to identify    the exact failed sync tasks and error messages.
2. Confirmed disk usage: df -h /var/lib/pulp — showed 100% usage.
3. Identified old content view versions that were no longer in use:
   hammer content-view version list --organization 'Default Organization'.
4. Deleted 8 old content view versions to reclaim space:
   hammer content-view version delete --id <id> --organization 'Default Org'.
5. Ran Pulp orphan cleanup to remove unreferenced content:
   foreman-rake katello:delete_orphaned_content.
6. Reclaimed 85 GB; partition now at 52% utilization.
7. Cleared the corrupted metadata by removing the affected repository directory    under /var/lib/pulp/content/yum and re-initiating the sync.
8. Triggered a fresh sync for RHEL 8 BaseOS and AppStream repositories from the    Satellite WebUI — both completed successfully.
9. Published a new content view version and promoted it to the Production lifecycle    environment for use in the upcoming patching cycle.
10. Expanded the /var/lib/pulp LVM volume by 200 GB using an available disk and     set up a Nagios alert at 70% threshold to proactively flag space issues.

PREVENTIVE MEASURES:
Automated weekly cleanup of old content view versions via a Satellite cron task.
